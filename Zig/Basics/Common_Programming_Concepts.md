# Zig 的基本编程概念
这一章，咱们来聊聊用 Zig 编程时必须知道的一些“基本零件”。

就像学英语要先学字母和单词一样，学任何编程语言，都得从变量、数据类型、函数这些最基础的概念开始。这些东西不是 Zig 独有的，但我们会看看它们在 Zig 里长什么样，有什么特别的玩法。

**具体来说，你会学到：**

1.  **常量与变量 (Constants and Variables):**
    *   **大白话：** 就像给一个数字或一段文字起个名字。比如，你可以说 `x` 就是 `5`。在 Zig 里，我们用 `const` 给一个值起个“永久”的名字（常量），用 `var` 起一个可以“随时更换”所代表的值的名字（变量）。
2.  **基本数据类型 (Basic Data Types):**
    *   **大白话：** 就是你存的数据是“什么种类”的。是整数（比如 `10`）、带小数点的数（比如 `3.14`）、单个字符（比如 `'A'`）、还是简单的“是”或“否”（`true` 或 `false`）。告诉 Zig 数据类型，它才知道该怎么处理。
3.  **函数 (Functions):**
    *   **大白话：** 就是把一堆操作打包，然后给它起个名字。比如你定义一个叫“泡茶”的函数，里面写清楚了烧水、放茶叶、倒水等所有步骤。以后想泡茶，直接喊一声“泡茶”（调用函数）就行了，不用每次都重复那些步骤。我们之前在 `helloworld.md` 里见过的 `main` 就是一个特殊的函数，是程序的入口。
4.  **注释 (Comments):**
    *   **大白话：** 就是你在代码旁边写的笔记，是给人看的，Zig 编译器会直接忽略它们。用来解释某段代码是干嘛的，或者提醒自己一些注意事项。

**这些概念是基石：**

把这些基本概念学扎实了，就像打好了地基，后面学习更复杂的 Zig 特性（比如错误处理、Comptime、指针等）就会轻松很多。可以说，你写的每一个 Zig 程序，都离不开这些基础。

**关于“关键字 (Keywords)”：**

*   **大白话：** Zig 语言自己保留了一些“**特殊词汇**”，这些词有特定的语法含义，告诉编译器要做什么事情。比如我们已经见过的 `const`, `var`, `fn`, `pub`, `try`, `@import` 等等。
*   **重要规则：** 因为这些词是 Zig 的“内部指令”，所以你**不能**用它们来给你的变量或函数起名字。比如，你不能创建一个叫 `const` 的变量：`var const = 5;` 这样是错误的。

总之，这一章就是带你熟悉 Zig 的基本“语法零件”，为后续的学习打好基础。
