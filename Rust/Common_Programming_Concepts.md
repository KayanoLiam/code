# 基本编程概念 (Common Programming Concepts)
这一章要讲的核心内容，用大白话解释一下。

这一章要讲的是**编程里最基本、最常见的一些概念**。你可以把它们想象成学任何一门语言（不管是中文、英文还是 Rust）都必须先掌握的“**字母、单词和基本语法规则**”。

几乎所有的编程语言，不管看起来有多不一样，骨子里都有一些共通的“零件”和“规则”。这一章就是要带你认识这些在 Rust 里是怎么体现的。虽然这些概念（比如变量、数据类型、函数等）不是 Rust 独有的，但我们会结合 Rust 的特点和习惯用法来讲解。

**具体来说，你会学到：**

1.  **变量 (Variables):**
    *   **大白话：** 就像你在草稿纸上写个代号（比如 `x`），然后让它代表一个数字（比如 `5`）或者一段文字。之后你就可以用 `x` 来代替那个数字或文字了。在程序里，变量就是给内存里的数据起个名字，方便我们存取和使用。
2.  **基本类型 (Basic Types):**
    *   **大白话：** 就是你存的数据大概是“什么种类”的。比如，是整数（像 `10`）、带小数点的数（像 `3.14`）、文字（像 `"你好"`）、还是简单的“是”或“否”（`true` 或 `false`）。告诉电脑数据的类型，它才知道怎么正确地处理。
3.  **函数 (Functions):**
    *   **大白话：** 就像给一连串的操作步骤起了个名字。比如你定义一个叫“做红烧肉”的函数，里面写清楚了所有步骤。以后每次想做红烧肉，直接喊一声“做红烧肉”（调用函数），程序就知道该怎么做了，而不用每次都重复写那些步骤。我们之前写的 `main` 就是一个特殊的函数，是程序的入口。
4.  **注释 (Comments):**
    *   **大白话：** 就是你在代码旁边写的笔记，是给人看的，电脑会自动忽略它们。用来解释某段代码是干嘛的，或者提醒自己一些注意事项。
5.  **控制流 (Control Flow):**
    *   **大白话：** 控制程序执行的“流程”或“路线”。代码不总是从上到下一行行执行的。有时候需要做判断（比如 “**如果** 用户猜对了，就说‘你赢了’，**否则** 就说‘猜错了’” - 这就是 `if/else`），有时候需要重复做一件事（比如 “**循环** 提示用户输入，直到猜对为止” - 这就是 `loop`）。控制流就是用来决定代码执行顺序和逻辑分支的。

**这些概念是基石：**

把这些基本概念学扎实了，就像打好了地基，后面学习更复杂的 Rust 特性（比如所有权、结构体、枚举等）就会更有底气。可以说，你写的每一个 Rust 程序，都离不开这些基础。

**关于“关键字 (Keywords)”：**

*   **大白话：** Rust 语言自己保留了一些“**特殊词汇**”，这些词有特定的语法含义，告诉编译器要做什么事情。比如我们已经见过的 `fn` (定义函数), `let` (声明变量), `mut` (表示可变), `use` (导入模块), `match` (模式匹配), `loop` (循环), `break` (跳出循环), `continue` (跳到下次循环), `true`, `false` 等等。
*   **重要规则：** 因为这些词是 Rust 的“内部指令”，所以你**不能**用它们来给你的变量或函数起名字。比如，你不能创建一个叫 `let` 的变量：`let let = 5;` 这样是错误的。
*   **未来保留：** 有些关键字目前可能还没用到，但 Rust 语言设计者觉得将来可能会用到，所以也提前“预定”了，你也不能用它们。附录 A 会有一个完整的列表。

总之，这一章就是带你熟悉 Rust 的基本“语法零件”，为后续的学习打好基础。
